# 彩叶 Agent 工具与技能架构（v2.3）

## 1. 目标与边界

本版本把方案 A、B、C 合并为一个受控的本地工具平台。模型只提出工具调用，Windows 客户端负责参数校验、权限确认、执行、结果截断和审计状态；模型不能直接执行 Shell、任意文件写入、自动发送邮件或模拟桌面点击。

本次范围包含 `T01-T12`（不含未定义的 T13-T19）、`T20-T21`、`T23-T24`、`T26-T27`，以及 `S01-S07`、`S09-S10`、`S12`。明确排除 `T22` 和 `T30-T34`。

## 2. 模块边界

| 模块 | 责任 | 不负责 |
|---|---|---|
| `AgentTools.cs` | 注册表、参数校验、权限会话、Skill 目录、SSRF/路径防护 | 具体工具业务、模型协议、主窗口布局 |
| `AgentToolExecutors.cs` | 18 个工具执行器、受限网络/文件操作、文档解析与本地工具存储 | 厂商协议、确认 UI |
| `ModelProviders.cs` | OpenAI Chat/Responses、Anthropic、Gemini、Cohere 工具循环协议 | 具体工具业务、用户确认 UI |
| `AgentDesktop.cs` | 工具会话编排、轻量状态反馈、图片选择、提醒展示和内存同步 | 解析供应商响应 |
| `ToolCenterForm.cs` | 工具中心与统一单次确认浮窗 | 执行工具或保存业务数据 |
| `SettingsStore` | 用户开关、白名单、搜索 Key 的原子与加密保存 | 工具执行 |
| QA Harness | 协议契约、权限、边界、持久化和视觉回归 | 真实外部账号写入 |

## 3. 执行流程

1. 主窗口构造系统提示、对话历史和当前启用的工具定义。
2. 协议适配器把统一 JSON Schema 转换为对应厂商格式。
3. 模型返回零个或多个工具调用。
4. 注册表验证工具名、必填字段、类型、长度、枚举、URL 和路径边界。
5. 涉及写入、删除、剪贴板、媒体控制、应用启动或知识库索引时，显示统一确认浮窗。
6. 工具在超时、结果大小和调用轮数限制内运行。
7. 结果以对应厂商的 `tool_result` / `function_call_output` 格式回传。
8. 最多循环 4 轮；最终仍输出彩叶 Agent 的中文显示文本、日语语音台词和情绪 JSON。

## 4. 权限模型

- `ReadOnly`：计算、时间、记忆检索、联网检索、授权目录内搜索和文档读取，可自动执行。
- `ConfirmEveryTime`：记忆写入/删除、提醒与日程写入、知识库索引、剪贴板读取、媒体控制和应用启动，每次确认。
- `Denied`：邮件发送、Shell、任意文件删除、浏览器登录/支付、自动发帖。
- URL 只允许 HTTP/HTTPS；拒绝内嵌凭据、回环、私网、链路本地、组播和云元数据地址。
- 本地文件只允许位于用户设置的规范化目录内，拒绝符号链接逃逸和相对路径穿越。
- 搜索 Key 使用 Windows CurrentUser DPAPI，与模型 Key 一样不进入提示词、记忆、日志和发行包。

## 5. 数据与恢复

- `settings.json`：工具总开关、A/B/C 开关、搜索服务、加密 Key、目录与应用白名单、技能开关。
- `memory.json`：沿用现有长期记忆格式，工具通过同一个原子存储入口读写。
- `reminders.json`、`calendar.json`、`email-drafts.json`、`knowledge-base.json`：独立原子 JSON 文件，损坏时从 `.bak` 恢复。
- 工具结果不持久化完整网页或剪贴板；知识库仅在用户确认后保存受限文本块和来源。

## 6. 可靠性限制

- 每个模型回答最多 4 轮工具调用，每轮最多 8 个调用。
- 每个参数按用途设置独立长度上限（例如路径 2,048 字符、邮件正文 12,000 字符），单个工具结果不超过约 48 KiB，网页下载不超过 1 MiB。
- 网络工具设置独立超时；一个工具失败以结构化错误返回，不让整个客户端崩溃。
- 多个写操作按顺序确认和执行，避免并发覆盖本地 JSON。

## 7. 扩展约定

新增工具必须同时提供：唯一名称、中文说明、JSON Schema、风险等级、分类、执行器、结果上限和 QA 用例。新增模型协议只实现统一会话与工具定义转换，不得绕过注册表或权限层。

## 8. 验收门槛

- 五类协议的工具请求、工具结果回传和最终文本解析均有离线契约测试。
- SSRF、路径逃逸、未授权副作用、超轮数和结果截断均有负向测试。
- 工具中心在 100% 与高 DPI 尺寸下无文字遮挡或系统默认硬边框。
- 原有模型连接、语音、长期记忆、保存/清空聊天与视觉小说界面回归全部通过。
