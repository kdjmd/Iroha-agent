# Iroha Agent v2.2.1

> 安全补充：从旧版明文配置迁移时，程序会同时清理或替换 `settings.json.bak` 与 `settings.json.corrupt`，避免旧侧文件继续保留 API Key。

> 界面补充：中等宽度和高 DPI 屏幕改用紧凑布局，完整布局按 `1672×941` 参考尺寸执行截图回归；名字牌、快捷操作和压缩状态不再截断。

## 主要更新

- 所有模型厂商 API Key 改为 Windows DPAPI `CurrentUser` 加密保存。
- 旧版明文 `settings.json` 在当前用户首次成功读取后自动迁移为密文。
- 密文复制到其他电脑或其他 Windows 用户后安全失效，并在设置中提示重新填写，不导致应用崩溃。
- 携带 API Key 的远程 HTTP 地址会被拒绝；本机 `127.0.0.1` 的 Ollama、LM Studio 等接口不受影响。
- API 地址禁止内嵌用户名和密码；厂商错误文本会移除活动 Key 和常见 Token 格式。
- 单次模型响应缓冲限制为 8 MiB，减少异常接口导致的内存风险。
- 修复 Cohere 聊天使用 `/v2/chat` 时，模型发现应请求 `/v1/models` 的兼容性问题。
- 新增 Tools 与 Skills 候选设计文档；本版本尚未接入任何 Tool。

## 验证

- 当前用户 DPAPI 加密、解密和多厂商 Key 切换通过。
- 旧明文自动迁移与损坏密文安全失效通过。
- 发布设置文件中不存在测试 Key 明文。
- 远程 HTTP、URL 内嵌凭据和错误脱敏回归通过。
- 模型、功能、记忆和语音部署完整回归通过。

发布附件不包含 API Key、聊天记录、长期记忆、用户设置或崩溃日志。
