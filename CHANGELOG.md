# Changelog

## 2.1.1 - 2026-07-17

- 修复部分 Windows 同时存在 `Path` / `PATH` 时，旧版 .NET 修改子进程环境导致 GPT-SoVITS 无法启动的问题。
- 为 Numba 使用独立可写缓存目录，避免 `librosa` 在只读运行时目录创建缓存时长期阻塞。
- 推理 YAML 自动复制到应用托管目录，GPT-SoVITS 可安全回写，同时保持外部运行时和源配置不变。
- FullVoice 部署改为完整性检查、磁盘空间检查、临时目录解压、就绪标记和失败自动恢复，不再先删除旧运行时。
- 修复分卷缺失、部署中断、跨 Windows 用户路径失效和端口占用时的错误恢复与反馈。
- 健康检查只接受真实 `/tts` OpenAPI，使用无代理短探测和墙钟超时；Python 诊断改为 UTF-8 非缓冲读取。
- 长期记忆改为原子写入、备份恢复、并发保护、去重与数量上限；普通一次性任务和 API 密钥不再误记。
- 修复会话右键菜单关闭时过早释放导致的重复错误弹窗；同一次运行的全局异常提示只显示一次。
- 增加记忆、跨电脑部署、只读配置、缺失分卷、动态端口和真实语音端到端回归。

## 2.1.0 - 2026-07-16

- 增加首次启动 GPT-SoVITS 自动发现、完整运行时解压、模型导入和配置匹配。
- 增加实时语音部署进度浮窗，展示检查、解压、模型导入、启动与完成状态。
- 增加设置内“重新部署语音”，严格限制为应用托管目录，不修改源语音包和外部运行时。
- 语音匹配结果写入本地设置，后续启动直接连接，不重复部署。
- 自动生成的 GPT-SoVITS 配置默认使用 CPU 全精度，提升跨显卡、驱动和无独显电脑的兼容性。
- 服务状态卡与语音控制区显示自动匹配状态。
- 增加部署安全、重新部署、进度 UI 和完整运行时语音 QA。
- 增加 Windows Portable 与 FullVoice 分卷发布脚本和 SHA-256 清单。
- 仓库正式收敛为 Windows-only，不再包含 Android 工程或 APK 发布流程。

## 2.0.0 - 2026-07-16

- 重做 Windows 主界面为视觉小说式布局。
- 增加大尺寸角色舞台、VN 对白框、快捷动作和玻璃质感工具栏。
- 增加会话新建、重命名、删除和置顶。
- 增加 Flash / Pro 模型徽标同步。
- 改进长期记忆、提示词压缩与角色回复设定。
- 改进 GPT-SoVITS WAV 校验、音量处理和播放状态反馈。
- 增加自然眨眼、口型与多种情绪逐帧动画。
